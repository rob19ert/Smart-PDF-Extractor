# Отчет по проекту: Smart PDF Extractor (Metallurgy & Engineering)

## 1. Описание и цели проекта
Проект представляет собой специализированный AI-инструмент для автоматического извлечения, кластеризации и классификации структурированных данных из технических PDF-документов. Основной фокус сделан на обработке металлургической и инженерной документации, где часто встречаются сложные формулы и специфические термины.

Цель инструмента — преобразовать неструктурированные PDF-страницы в размеченные данные (текст, таблицы, физические/математические формулы) с возможностью визуальной проверки и экспорта в Excel.

## 2. Архитектура и используемые технологии
Проект построен на модульной архитектуре с использованием современных библиотек для работы с документами и машинного обучения:

*   **Извлечение базовых элементов (unstructured[pdf])**: Используется стратегия `hi_res` для высокоточного распознавания базовых блоков (текст, таблицы) на странице.
*   **Графовая кластеризация**: Специальный алгоритм на основе обхода графа в глубину (DFS), который объединяет разрозненные слова и строки в единые осмысленные абзацы на основе их пространственной близости и выравнивания.
*   **Машинное обучение (scikit-learn)**: Обученная модель Logistic Regression (`formula_classifier.pkl`) с TF-IDF векторизацией для первичной классификации формул. Модель дополнена структурным анализатором (взвешивание контекста, распознавание LaTeX-символов).
*   **Визуализация (pdfplumber, matplotlib)**: Генерация изображений (PNG) с наложенными цветными рамками (Bounding Boxes), соответствующими типам извлеченных блоков (синий — текст, зеленый — формулы, красный — таблицы).

## 3. ИИ-Инспекция с использованием OpenRouter
Для повышения точности работы парсера в проект интегрирована логика "ИИ-Инспектора" с использованием бесплатных LLM через платформу **OpenRouter**.

*   **Используемая модель**: `poolside/laguna-m.1:free`. Модель выбрана благодаря ее способности хорошо понимать структурированные текстовые данные (JSON) и логику классификации.
*   **Принцип работы**: После первичного извлечения данных, парсер собирает текстовое содержимое блоков страницы и отправляет его модели с инструкцией найти ошибки (например, когда сложная физическая формула ошибочно определена как обычный текст, или наоборот).
*   **Самообучение**: При нахождении ошибок парсер автоматически применяет исправления, перерисовывает визуальные графики и сохраняет исправленные данные в файл `retrain_dataset.csv` для последующего дообучения ML-модели.
*   **Безопасность конфигурации**: Ключи доступа (включая `OPENROUTER_API_KEY`) безопасно загружаются из локального файла `.env` с помощью библиотеки `python-dotenv`, что соответствует стандартам архитектуры.

## 4. Результаты работы
На выходе программа генерирует:
1.  **Excel-отчеты** (`*_blocks_final.xlsx`) — содержат полные извлеченные данные с привязкой к страницам и типам контента.
2.  **Размеченные изображения** в папке `output_images/` — для быстрой визуальной верификации качества парсинга.

Тестовые прогоны показали успешную работу графового алгоритма слияния и высокую эффективность модели `poolside/laguna-m.1:free` в обнаружении и корректировке огрехов структурного парсера.
