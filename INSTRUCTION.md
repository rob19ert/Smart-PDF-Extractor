# Smart PDF Extractor: Жизненный цикл и Инструкции

Специализированный инструмент для извлечения, кластеризации и классификации данных из технических PDF-документов (металлургия и инженерия).

---

## 1. Жизненный цикл проекта (Project Lifecycle)

Процесс работы с инструментом разделен на несколько ключевых этапов:

### Этап 1: Подготовка и конфигурация
- **Настройка окружения**: Установка Python-зависимостей и системных библиотек (Poppler, Tesseract).
- **Конфигурация API**: Настройка ключей доступа в файле `.env` для работы с OpenRouter (ИИ-инспектор) и Google Gemini (финальный анализ).

### Этап 2: Загрузка и первичная обработка
- **Размещение данных**: Копирование целевых PDF-файлов в корневую директорию проекта.
- **Парсинг (Partitioning)**: Использование библиотеки `unstructured` со стратегией `hi_res` для разбиения PDF на базовые элементы (текст, таблицы, заголовки).

### Этап 3: Структурная обработка и Классификация
- **Графовое слияние**: Применение алгоритма на основе поиска в глубину (DFS) для объединения разрозненных текстовых блоков в связные абзацы.
- **Гибридная классификация**:
    - **ML-слой**: Определение типа формул (физика/химия/мат) через `LogisticRegression`.
    - **Эвристический слой**: Проверка химических элементов, LaTeX-символов и контекста.

### Этап 4: ИИ-Инспекция и Валидация
- **Верификация через LLM**: Отправка структуры страницы ИИ-инспектору (через OpenRouter) для поиска ошибок классификации.
- **Автоматическая коррекция**: Применение исправлений "на лету", перерисовка визуальных схем и сохранение данных для дообучения в `retrain_dataset.csv`.

### Этап 5: Генерация результатов и Аналитика
- **Экспорт в Excel**: Сохранение структурированной таблицы элементов.
- **Визуализация**: Генерация PNG-файлов с цветовой разметкой блоков (Bounding Boxes) в папке `output_images`.
- **Интеллектуальный отчет**: Создание Markdown-файла с кратким содержанием и выводами по документу с помощью Gemini.

### Этап 6: Поддержка и Обучение
- **Дообучение модели**: Периодический запуск `train_metallurgy.py` для обновления весов классификатора на основе накопленных правок от ИИ-инспектора.

## 2. Инструкции по запуску (Run Instructions)

### Предварительные требования
1. **Python 3.12+**
2. **Системные зависимости**:
   - Windows: Установите [Poppler](https://github.com/oschwartz10612/poppler-windows/releases) и [Tesseract OCR](https://github.com/UB-Mannheim/tesseract/wiki). Добавьте пути к `/bin` в системную переменную `PATH`.
   - Linux: `sudo apt-get install poppler-utils tesseract-ocr`

### Шаг 1: Установка зависимостей
```bash
pip install -r requirements.txt
```

### Шаг 2: Настройка переменных окружения
Создайте файл `.env` в корневой директории и добавьте туда ваши ключи:
```env
OPENROUTER_API_KEY=ваш_ключ_openrouter
GOOGLE_API_KEY=ваш_ключ_gemini
```

### Шаг 3: Запуск обработки
**Для обработки всех PDF в папке:**
```bash
python main.py
```

**Для обработки конкретного файла:**
```bash
python main.py имя_файла.pdf
```

### Шаг 4: Обучение классификатора (Опционально)
Если вы добавили новые данные или хотите обновить модель:
```bash
python train_metallurgy.py
```

## 3. Анализ результатов

После завершения работы скрипта проверьте:
1. **Папка `output_images/`**: Визуальное подтверждение точности.
   - **Синий**: Текст
   - **Зеленый**: Формулы
   - **Красный**: Таблицы
2. **Файлы `*_blocks_final.xlsx`**: Полный набор данных для работы в Excel.
3. **Файлы `*_gemini_analysis.md`**: Текстовое резюме документа.
